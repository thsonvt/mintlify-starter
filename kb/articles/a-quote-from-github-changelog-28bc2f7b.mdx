## Summary

GitHub Changelog introduces a new default cooldown period for Dependabot, requiring a three-day wait after a new release before opening version update pull requests.

## Key Insights

> "Dependabot now waits until a new release has been available on its registry for at least three days."
>
> — Describing the new default behavior of Dependabot regarding version updates.

> "This cooldown is now the default and requires no configuration."
>
> — Highlighting the ease of use for developers with the new Dependabot feature.

## Full Article

14th July 2026 Dependabot now waits until a new release has been available on its registry for at least three days before opening a version update pull request. This cooldown is now the default and requires no configuration. — GitHub Changelog, embracing dependency cooldowns
