### no cap, how does this work?

The rizzler is a [low-level merge driver](https://git-scm.com/docs/merge-config?ref=ghuntley.com) that can be used as a command-line tool without Git and can be configured into Git itself as a resolver strategy. It queries the LLMs you have configured (there's a bundle of strategies within, and I'd [love for folks to send in more](https://github.com/ghuntley/rizzler/blob/trunk/CONTRIBUTING.md?ref=ghuntley.com#-where-we-really-need-your-help-resolution-strategies-))

![](https://ghuntley.com/content/images/2025/04/Untitled-diagram-2025-04-23-115511.png)

If a file hits eight merge conflicts and can't crack one, it'll tackle the rest and send an "oops" back to Git, halting the merge party. That's your cue to dust off your favourite editor and resolve that stubborn conflict manuallyâ€”throwback style, like it's 2005. On the bright side, successful fixes get cached on disk, cutting down on LLM costs and giving you a productivity boostâ€”no more twiddling thumbs waiting for the LLM to chime in!
